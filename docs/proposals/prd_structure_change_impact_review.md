## 2. 已解决的问题

1. PRD 一级标题 `一句话摘要` 已改为 `摘要`。
2. 自动生成 PRD 的一级标题也同步改为 `摘要`。
3. `目标 / 非目标` 已从前置位置移到埋点之后。
4. `成功指标` 已跟随 `目标 / 非目标` 放在其后。
5. 模板和生成器的章节编号已改成连续编号。

```text
6. 范围定义
7. 方案概述
8. 详细需求
9. 需求明细表
10. 用户故事与验收标准
11. 异常、边界与兼容性
12. 非功能要求
13. 埋点与数据方案
14. 目标 / 非目标
15. 成功指标
16. 依赖、风险与开放问题
17. 上线与灰度方案
18. 验收 Checklist
19. 版本记录
20. 附录 / 链接资料
```

## 4. 保留还是回退建议

### 建议保留

- `一句话摘要` 改为 `摘要`。
- `目标 / 非目标` 与 `成功指标` 移到埋点之后。
- 章节连续编号。
- 自动生成器同步章节顺序。

理由：这是用户明确确认过的结构调整。

### 暂不继续扩展

- 暂不继续改 `prototype_wireframes`。
- 暂不新增 `page_flow.md`、`page_specs.md` 默认生成。
- 暂不改 schema。
- 暂不新增 harness。

理由：页面说明/跳转关系机制虽然已经有方案，但用户还没有批准写入长期模板和生成链路。

### 需要用户审核后决定

- 目标/指标放到第 14、15 章后，阅读体验是否可接受。
- `埋点与数据方案` 是否应该继续排在目标/指标之前。
- 是否要把 `目标 / 非目标` 改成更靠近结尾的“目标复核 / 范围边界”。
- 是否正式批准页面说明/跳转关系机制进入长期模板。

## 5. 本轮测试方式

用当前生成链路生成一个“打车产品 PRD”测试样例，不手写美化。

测试重点：

1. 目录是否按新顺序输出。
2. `摘要` 是否生效。
3. `目标 / 非目标` 和 `成功指标` 是否移动到埋点之后。
4. 是否仍出现旧的 `原型图 / 线框图` 问题。
5. 页面说明/跳转关系是否仍缺失。
6. 生成内容是否因为模板调整变得更难读。

先不要再改稳定链路。先让用户审核测试 PRD，确认：

- 章节顺序是否真的更符合阅读习惯。

- 页面说明/跳转关系机制是否要正式进入模板和生成器。
