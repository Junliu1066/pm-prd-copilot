## 关键判断

### 1. 市场默认值

`skill-update-20260424T041400Z` 里提出“如果用户未指定市场，从请求、项目缓存或用户偏好读取市场默认值”。

我的建议：暂不写进通用 Skill 的硬规则。后续只允许按这个顺序解析：

1. 当前请求明确指定。
2. 当前项目已批准的项目偏好缓存。
3. 已批准的长期偏好。
4. 仍不明确时标记为 `unresolved` 或向用户确认。

效果：避免把“中国市场”或任何市场写死到通用 Skill，也避免跨项目误用。

### 2. 竞品外延

该补丁提出竞品分析要覆盖直接工具、相邻平台、平台原生替代、内容社区、线下服务、手动作业等。

我的建议：方向正确，但先保持 candidate。下一次处理 PRD analysis suite 时，可以把它做成“竞品影响地图”的候选输出，而不是强制每个项目都列满所有类别。

效果：竞品判断会更完整，但不会让小项目 PRD 被过度分析拖慢。

### 3. PRD 分析视觉化

该补丁提出用户群体矩阵图、方案 / Skill / MCP 分工流程图。

我的建议：不直接按原文应用。A1 已经确认 PRD 稳定底线是页面说明、页面跳转关系、原型图层；其他图表按项目必要性放在对应章节。Skill / MCP 分工图属于内部治理或复杂方案审查，不应该默认塞进所有 PRD。

效果：保留“可视化辅助理解”的价值，同时避免 PRD 再次退化成集中式图表堆叠。

### 4. 原型可见交付

`skill-update-20260425T000000Z` 的方向已经和当前规则一致：PRD 阶段需要原型图层，PNG / HTML / UI 在用户确认后进入后续阶段。

我的建议：继续保留规则，但不提交 fitness 项目缓存，不把 fitness 的具体表达转成长期默认。

效果：后续原型更容易被你直接审核，同时不会污染别的项目。
