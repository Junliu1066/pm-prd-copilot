# 0-1 普通业务 PRD 结构样例

- 状态：golden case candidate。
- 来源：从一次 0-1 普通业务 PRD 测试项目中抽象，不保留项目 ID、绝对路径、测试目的或项目过程材料。
- 用途：验证 PRD 主链路是否保持稳定结构底线。
- 不用途：不代表所有 PRD 的唯一写法，不作为 UI 设计稿，不作为项目交付物。

## 为什么保留

该样例覆盖了普通业务 MVP PRD 中容易退化的结构点：

- 图表必须放在对应章节，不能集中堆成 `PRD 可视化层`。
- PRD 必须包含页面说明、页面跳转关系和原型图层。
- PRD 阶段只提供页面级低保真说明，不默认输出 PNG、HTML 或高保真 UI。
- 普通非 AI 项目不能强行出现 AI 模型选型。
- 测试目的、文档目的和产品业务目标必须分开，不能混写。

## 文件

| 文件 | 作用 |
|---|---|
| `input.md` | 脱敏后的普通业务 MVP 输入。 |
| `expected_prd.md` | 脱敏后的结构参考 PRD。 |
| `acceptance.md` | 回归检查和人工审核要点。 |

## 使用边界

该样例可以进入 regression 作为结构检查基线，但仍是单个候选样例。正式 golden sample portfolio 需要后续再收集不同类型 PRD 后统一升级。
