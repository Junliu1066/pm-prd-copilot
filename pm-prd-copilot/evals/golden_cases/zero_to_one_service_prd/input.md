# 0-1 本地服务履约平台 MVP 输入样例

## 产品背景

面向本地即时服务场景，用户需要快速发起服务请求，服务提供者需要接收订单并完成履约，平台需要提供订单状态、价格预估、消息通知、支付评价和客服兜底。

## 一期范围

- 用户端：发起服务请求、确认信息、等待接单、查看履约状态、支付评价。
- 服务端：上线接单、查看订单、更新到达 / 开始 / 完成状态。
- 平台端：基础派单、订单状态机、消息通知、价格预估、支付记录。
- 后台端：订单查询、客服处理、基础运营指标和日志。

## 暂不做

- 预约服务。
- 拼单或多人共享订单。
- 企业账户。
- 会员体系。
- 复杂动态定价。
- 跨城市服务。

## 关键问题

- 如何降低用户等待不确定性？
- 如何让服务提供者稳定接单并按状态履约？
- 如何处理取消、超时、支付失败和安全异常？
- 如何让客服快速定位订单并介入？
