# 内部治理嵌入规则

当用户明确说明项目是自有项目、内部项目或可信核心团队使用时，Codex 开发文档必须嵌入完整治理系统。

## 必须包含

- facts / assumptions / open questions。
- P0 / P1 / P2 门禁。
- Capability Enablement。
- Skill/MCP Discovery。
- 内部多管家组织系统。
- 效率部。
- 随机审计。
- 教师 / 学习沉淀。
- harness 检查。
- Codex 任务包和写入边界。
- 人工确认点。

## 禁止

- 不把内部治理写进用户侧 PRD。
- 不把内部治理写进 B 包。
- 不绕过人工确认修改稳定 Skill、memory、registry、harness 或模型供应商。
