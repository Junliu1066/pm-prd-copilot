# 开发文档通用规则

## PRD 输出

PRD 是产品文档，必须输出：

- 项目摘要。
- 背景与机会。
- Facts / Assumptions / Open Questions，并按 P0/P1/P2 分级。
- 目标用户和核心场景。
- 产品目标和成功指标。
- Scope / Out of Scope。
- 产品总览、MVP 范围、核心业务、页面信息架构、风险控制等辅助理解图，必须放在对应章节，不能集中堆到“PRD 可视化层”。
- 用户流程 / 业务流程，正文必须有流程图。
- 页面说明、页面跳转关系和原型图层，正文必须覆盖关键页面、入口、退出路径、权限、异常和页面级低保真原型说明；PNG / HTML / 高保真 UI 只在用户确认原型阶段后输出。
- 功能需求、状态、异常和验收标准。
- AI 模型选型仅在项目涉及 AI 时出现，涉及 AI 时必须写任务拆分、模型路由、fallback、成本/延迟/质量、评测和合规。
- 用户故事与验收标准。
- 数据与埋点。
- 权限与角色。
- 风险与边界。
- 发布计划。
- 验收 Checklist。

PRD 不写内部开发治理、内部工具、内部任务包和内部审计。

## Codex 开发文档输出

开发文档是执行文档，必须输出：

- 产品 / 开发边界。
- 输入文件。
- 开发流程。
- 分期交付路线：一期、二期、三期、最终。
- 每期目标、交付物、实现效果、验收标准。
- 工程任务拆分。
- 数据模型、API、AI 实现、前端页面、测试、部署。
- 任务包格式。
- 人工确认点。
- 打包与分发规则。

开发文档不默认重复 PRD 原型图层；需要引用 PRD 页面说明、页面跳转关系、已确认原型或已确认线框材料。

除非用户明确要求内部版，否则默认 B 执行口径。B 执行口径必须使用纯英文输出。
