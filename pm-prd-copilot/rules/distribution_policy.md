# 分发策略

## 目标

开发文档必须根据分发对象选择不同版本，既保证 Codex 开发能使用完整治理系统，也保护产品经理 Skill 的内部框架不被外部读者逆向。

## 分发模式

| 模式 | 使用场景 | 输出方式 |
|---|---|---|
| `A_INTERNAL_FULL` | 用户明确说这是自己的项目、内部项目、可信核心团队使用 | 允许嵌入完整治理系统 |
| `B_EXECUTION_PACK` | 给外部开发者、外包、合作方、非核心成员，或分发对象不明确 | 只暴露执行要求、质量门禁和验收标准 |

## 默认规则

- 用户明确说“内部版 / 我自己用 / 自己项目 / 我的项目 / 可信团队”：使用 `A_INTERNAL_FULL`。
- 其他所有情况默认使用 `B_EXECUTION_PACK`。
- 用户说“给别人 / 外包 / 合作方 / 对外 / 发给开发”：必须使用 `B_EXECUTION_PACK`。
- 分发对象不明确：默认使用 `B_EXECUTION_PACK`，并单独保留内部完整版。

## B 包命名规则

- 文件名使用字母代号，例如 `A.md`、`B.md`、`C.md`。
- 对外 zip 文件名使用 `B`，不要出现“外部、脱敏、内部完整版、治理系统”等提示。
- README 可以说明字母文件对应的交付物，但不能暴露内部框架名称。
- B 包内所有文件必须使用英文，包括 README 和所有字母文档。

## 禁止

- 不把内部治理系统写进 PRD。
- 不把内部治理系统写进 B 包。
- 不在 B 包中出现内部框架名称、角色体系、命令、注册表、学习机制或模板路径。
- `agent.md` 只能写中性执行规则，不能写完整内部组织结构。
