# B 包保护规则

B 包是给外部开发者、外包、合作方或不明确分发对象使用的开发执行包。

## 文件命名

- 使用字母代号。
- 根目录 README 只解释交付物，不解释为什么使用字母。
- 包名使用 `B`，不使用“外部”“脱敏”等字样。

## 内容表达

允许表达：

- 开发准备检查。
- 质量门禁。
- 独立质量抽查。
- 效率检查。
- 交付验收。
- 问题复盘。

禁止表达：

- 内部多管家结构。
- 效率部和教师的真实机制。
- Skill/MCP 路由。
- harness 命令。
- registry、steward、memory 体系。
- 内部模板和可复用规则路径。

## 交付门禁

B 包打包前必须运行泄漏检查。只要命中 `redaction_terms.yaml` 中的禁止词，打包失败。
