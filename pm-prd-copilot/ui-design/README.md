# UI Design Layer

This folder contains supervised UI design rules for high-fidelity prototype work.

The catalog is inspired by public, open-source UI design resources such as:

- `nextlevelbuilder/ui-ux-pro-max-skill`: design reasoning, style matching, color, typography, and UX guideline structure.
- `creativetimofficial/ui`: shadcn/Tailwind component and block catalog patterns.

Use this layer before building visual prototypes:

```text
PRD / brief
 -> UI style direction
 -> design tokens
 -> screen blueprint
 -> component map
 -> prototype
 -> screenshot review
```

## Rules

- Treat visual styles as candidates, not defaults.
- Pick a style only after checking product type, user task, usage frequency, information density, and trust requirements.
- Do not apply expressive styles such as concrete, brutalist, cyberpunk, or glassmorphism to dense operational software unless the user explicitly approves.
- Keep source attribution when adapting open-source style catalogs or component catalogs.
- Do not copy hidden prompts from closed systems.
