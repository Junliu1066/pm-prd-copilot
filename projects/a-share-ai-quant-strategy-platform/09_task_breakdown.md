# A股 AI 量化策略研究平台任务拆分

- 文档状态：Draft / 可派发任务版
- 适用读者：研发负责人、Codex 执行线程、代码集成人
- 最后更新时间：2026-05-08

---

## 1. 任务执行规则

每个任务必须写清：

- 目标。
- 输入文档。
- 可修改文件。
- 不可修改文件。
- 验收标准。
- 必跑检查。

合规底线：

- 不新增荐股、跟单、持仓、买卖点、实时信号、自动下单、券商账户。
- 不把积分描述为收益、返利、可提现资产。
- 量化能力默认 `research_only` / `simulation_only`。

## 2. `codex/p0-spring-api`

目标：建立 Spring Boot 基础工程和 P0 API 骨架。

可改：`backend/**`、`api/**`、`db/**`、`06_api_spec.md`、`08_backend_engineering_spec.md`。  
不可改：`prototype/html/**`、PRD 主范围。

### 任务

1. 初始化 Spring Boot 3.x 工程。
   - 验收：应用可启动，`/actuator/health` 返回 UP。
   - 检查：`./mvnw test`。
2. 实现统一响应、异常、traceId。
   - 验收：错误响应符合 `06_api_spec.md`。
   - 检查：MockMvc 覆盖 400/401/403/404。
3. 接入 Spring Security 和 JWT。
   - 验收：游客接口可访问，用户接口必须登录，后台接口必须管理员角色。
   - 检查：鉴权测试。
4. 建立 Flyway migration。
   - 验收：空 MySQL 可顺序建表。
   - 检查：migration 集成测试。
5. 接入 OpenAPI。
   - 验收：本地可访问 Swagger UI。
   - 检查：接口路径与 `06_api_spec.md` 对齐。

---

## 3. `codex/p0-quant-engine`

目标：实现市场温度、量化研究、回测仿真和风控计算基础能力。

可改：`backend/marketdata/**`、`backend/indicator/**`、`backend/strategyengine/**`、`backend/backtest/**`、`backend/risk/**`、对应 Spring package 路径和量化测试。  
不可改：积分账务、未授权 `db/migration/**`、C 端买卖信号、券商账户、自动下单。

### 任务

1. 实现数据批次管理。
   - 验收：可创建、查询 `data_batches`。
   - 检查：Repository + Service 测试。
2. 实现市场温度快照读取。
   - 验收：`GET /market/temperature/latest` 返回最新快照。
   - 检查：MockMvc 测试。
3. 实现指标快照计算入口。
   - 验收：后台可触发重新计算并写 `indicator_snapshots`。
   - 检查：计算服务单测。
4. 实现回测任务创建。
   - 验收：`runMode` 只接受 `research_only` / `simulation_only`。
   - 检查：拒绝 `live_trade` 的测试。
5. 实现风控检查。
   - 验收：最大回撤、异常收益、违规表达可写入 `risk_checks`。
   - 检查：风控规则单测。
6. 输出报告可用的回测摘要。
   - 验收：报告服务可读取年化、回撤、夏普、胜率、净值曲线。
   - 检查：集成测试。

## 4. `codex/p0-points`

目标：实现积分账户、测试充值、每日登录赠送、积分消耗和权益。

可改：`backend/points/**`、`backend/user/**`、`db/migration/**`、积分相关 API 文档。  
不可改：竞技场核心榜单、量化计算逻辑、HTML 原型。

### 任务

1. 创建积分账户。
   - 验收：用户首次登录后自动有账户。
   - 检查：用户创建集成测试。
2. 创建测试充值订单。
   - 验收：`1 元 = 1 积分`，P0 `paymentChannel=test` 直接入账。
   - 检查：金额、积分、流水测试。
3. 每日登录赠送。
   - 验收：每天最多赠送 1 积分。
   - 检查：重复提交不重复赠送。
4. 积分消耗。
   - 验收：余额不足返回 `POINTS_NOT_ENOUGH`，成功后生成权益。
   - 检查：并发扣减测试。
5. 幂等。
   - 验收：同 key 同请求返回同结果，不同请求返回 `IDEMPOTENCY_REPLAY`。
   - 检查：幂等测试。

## 5. `codex/p0-web-prototype`

目标：保持 HTML 原型为设计和开发对齐基线。

可改：`prototype/html/**`、`02_prototype_layer.md`。  
不可改：后端代码、数据库 migration。

### 任务

1. 检查页面覆盖。
   - 验收：首页、市场温度、策略、报告、积分、风险、后台、竞技场页面可访问。
2. 移除不需要的手机状态栏组件。
   - 验收：已被用户否定的状态栏不回归。
3. 保持积分制度表达。
   - 验收：不出现 C 端会员、月票、观众票、订阅作为收费制度。
4. 细化接口状态。
   - 验收：页面有空态、错误态、权限态、积分不足态。

检查：浏览器打开 `prototype/html/index.html`，无控制台错误。

2. 实现首页、市场温度、策略列表、策略详情、报告试看。

## 8. `codex/p0-admin`

目标：实现运营后台 P0 能力。

可改：`admin/**`、`backend/admin/**`、`backend/audit/**`。  
不可改：C 端主路径大范围重构。

### 任务

1. 后台登录和角色权限。
2. 策略管理：新建、编辑、提交审核、发布、下线。
3. 报告管理：列表、发布。
4. 审核任务：通过、驳回、修改建议。
5. 积分和充值记录查询。
6. 操作日志查询。

验收：

- 所有敏感操作写 `operation_logs`。
- 未审核内容不能发布。
- 后台接口需要管理员 token。

## 9. `codex/ai-compliance`

目标：实现 AI 输出结构、风险词和合规拦截。

可改：`backend/ai/**`、`backend/risk/**`、prompt 文档、测试。  
不可改：UI 大范围重构。

### 任务

1. 定义 AI 输出 DTO。
2. 定义风险词库和命中规则。
3. 实现输出拦截。
4. 写 `ai_output_logs`。
5. 命中高风险时写 `audit_tasks`。

验收：

- AI 不输出买入、卖出、持有、仓位、价格、跟单。
- 输出包含来源、置信度、风险提示。
- 命中风险词可追溯。

## 10. `codex/p1-arena`

目标：实现策略竞技场 S1 内测。

可改：`arena/**`、`backend/arena/**`、竞技场前端页面。  
不可改：P0 登录、积分核心账务、量化实盘能力。

### 任务

1. 赛季管理。
2. 选手报名和审核。
3. 排行榜：综合榜、今日涨幅、本周黑马、回撤榜、夏普榜。
4. 1v1 对决和投票。
5. 观察内容积分解锁。
6. 异常成绩取消和日志。

验收：

- 榜单异常可取消并记录原因。

## 11. `codex/qa-gates`

目标：建立自动化检查和验收门禁。

可改：`tests/**`、`scripts/**`、`automation/quality_gates.md`。  
不可改：产品功能实现。

### 任务

1. 文档一致性检查。
2. API 规格完整性检查。
3. 数据库 migration 检查。
4. Spring Boot 测试命令封装。
5. HTML 原型控制台检查。
6. 合规风险词检查。
7. `THREAD=<branch> make check-boundary` 文件边界检查。

验收：

- 检查脚本能在本地运行。
- 能发现废弃收费话术和高风险交易话术。
- 能发现 Codex 分线程越界修改。
- 能输出通过 / 失败 / 修复建议。
