# 失败处理协议

- 文档状态：Draft / SDD 执行协议
- 适用对象：Codex 分线程、集成负责人、QA、合规审核
- 最后更新时间：2026-05-10

---

## 1. 总原则

失败后先止损，再定位，再修复。

禁止：

- 失败后继续扩大修改范围。
- 为了通过检查删除门禁。
- 用覆盖方式解决冲突。
- 未审批修改 API、数据库、积分、量化边界或合规规则。
- 为保留工作量降低合并标准。

---

## 2. 测试失败

触发：

- `make check-all` 失败。
- `./mvnw test` 失败。
- `npm run lint` 或 `npm run build` 失败。
- `node --check` 失败。

处理：

```text
停止新增功能
-> 记录失败命令和输出摘要
-> 判断失败属于当前线程还是上游线程
-> 当前线程问题则最小修复
-> 上游问题则标记 blocked 并提交阻塞说明
-> 修复后重新运行同一命令
```

交付摘要必须写：

```text
失败命令：
失败原因：
修复动作：
复跑结果：
残余风险：
```

---

## 3. 合规失败

触发：

- `scripts/check-compliance.sh` 失败。
- 文案出现收益承诺、荐股、带单、跟单、自动下单、券商账户、买卖点、实时信号等正向能力描述。
- 积分被描述成收益、返利、可提现资产。

处理：

```text
停止发布或合并
-> 找到命中行
-> 判断是否为禁止语境
-> 如果不是禁止语境，改写为研究、风险提示或不做范围
-> 复跑 make check-all
```

如果无法判断是否合规，按不可上线处理。

---

## 4. API 漂移

触发：

- 实现接口与 `06_api_spec.md` 不一致。
- 前端临时发明字段。
- 后端返回未记录的错误码。
- request / response 字段改动未同步文档。

处理：

```text
停止联调
-> 以 06_api_spec.md 为准
-> 判断是否需要契约变更
-> 不需要变更则修代码
-> 需要变更则提交契约变更请求
-> 契约 owner 更新文档后再实现
```

不得先合并代码再补文档。

---

## 5. Migration 冲突

触发：

- 两个线程生成同一个 migration 版本号。
- migration 顺序与 `07_database_schema.md` 不一致。
- 建表 SQL 无法从空库执行。
- 积分、回测、审计表缺少关键约束。

处理：

```text
暂停相关线程
-> 集成负责人重排 migration 顺序
-> 保留线性版本号
-> 重新从空库执行
-> 更新 07_database_schema.md
-> 通知下游线程
```

禁止在不同线程里各自私改 migration 版本号。

---

## 6. 文件越界

触发：

- 修改了 Task Brief 未授权文件。
- 修改了 `14_file_boundary_matrix.md` 标记为 `read`、`locked`、`forbidden` 的路径。
- 未经 owner 修改契约文档。

处理：

```text
停止当前线程
-> 列出越界文件
-> 判断越界修改是否必须保留
-> 必须保留则提交变更请求
-> 非必须保留则回退本线程越界修改
-> 更新 Task Brief
```

不得回退用户或其他线程已有修改。

---

## 7. 线程失控

触发：

- 修改范围明显超过任务目标。
- 同时修改多个无关模块。
- 多次失败仍继续扩大实现。
- 引入实盘、自动交易、跟单、买卖信号等高风险能力。

处理：

```text
标记线程 blocked
-> 停止新增提交
-> 集成负责人审查 diff
-> 可保留部分摘取到新小任务
-> 其余标记 abandoned
```

失控线程不得直接合并。

---

## 8. 上下文污染

触发：

- 线程使用了过期 PRD 或旧收费制度。
- 把 raw source notes 当成当前事实。
- 把外部分享版内容写入内部工程规范。
- 忽略项目级 `AGENTS.md`。

处理：

```text
停止实现
-> 重新读取 AGENTS.md 和当前工程文档
-> 标记污染来源
-> 删除或改写受污染内容
-> 复跑文档和合规检查
```

当前事实以最新用户指令、PRD、内部开发文档和工程契约为准。

---

## 9. 交付失败

触发：

- 交付摘要缺少验证结果。
- 未说明未完成事项。
- 未说明风险。
- 未说明需要主线集成处理的问题。

处理：

```text
退回补交付摘要
-> 不进入合并
-> 补齐验证命令和结果
```

合格交付摘要格式：

```text
完成内容：
修改文件：
验证结果：
未完成事项：
风险：
需要主线集成处理：
```
