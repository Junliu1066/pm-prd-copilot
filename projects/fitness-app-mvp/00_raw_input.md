# 健身APP MVP

## 原始需求
- 我现在要做一个健身 APP。
- 核心功能需要包含：
  - 食物热量拍摄计算
  - 组间歇闹钟

## 当前理解
- 产品方向：健身 / 减脂 / 训练辅助类 APP
- 当前明确的核心价值有两类：
  - 用户通过拍照识别食物并估算热量，降低记录饮食门槛
  - 用户在力量训练或间歇训练过程中使用组间歇闹钟，规范休息时长

## 已知未确认项
- 目标用户更偏向减脂人群、增肌人群，还是泛健身人群
- 首发平台是 iOS、Android，还是双端同时做
- 食物热量计算是只做拍照估算，还是还要支持手动修正、份量调整、历史记录
- 组间歇闹钟是简单倒计时，还是要支持训练计划联动、动作切换、语音提醒、后台运行
- 是否需要账号体系、训练记录、饮食记录、体重记录、会员付费等基础能力
- 第一期 MVP 只做这两个功能，还是要形成一个完整首页/训练页/饮食页闭环
