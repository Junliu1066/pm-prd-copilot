## 1. 产品想法

我们准备做一个 GEO 服务 / 平台，帮助不同行业的公司提高在 AI 检索、AI 问答和 AI 推荐结果里的曝光率、品牌提及率、品牌推荐率和竞品对比优势。

首期不是先做完整 GEO SaaS 平台，而是先把 GEO 服务包装成一套可销售、可交付、可复购、可逐步 Agent 化放大的服务产品。

第一阶段产品形态：

- GEO 免费检测
- GEO 初筛结果
- 顾问解读
- GEO 体检报告
- 客户可溯源数据看板
- 后续优化 / 代运营转化

内部 Agent 和工作台是交付效率工具，不是 MVP 对客户销售的核心产品。

## 2. 已确认事实

- 我们已经做过 GEO 服务 MVP 实验。
- 同行已经为 GEO 服务买单，存在真实付费信号。
- 市面同行常见路径是 Web 端引流到私域，在私域售卖服务，并在服务期间给客户可溯源的数据看板做实时反馈。
- 客户核心关心 AI 是否推荐自己、是否推荐竞品、是否正确理解和描述自己的品牌、产品和服务。
- 服务不是卖 GEO 技术概念，而是帮助企业抢占 DeepSeek、豆包、Qwen、元宝等 AI 检索 / AI 问答入口。
- 第一阶段不先做复杂平台，先跑通 GEO 诊断报告服务。
- 推荐商业路径为免费检测获客、报告成交、优化项目赚钱、月度代运营复购、Agent 降本。
- 风险边界明确：不能承诺一定被 AI 推荐，不能承诺排名提升，不能操控 AI 答案，不能做虚假评价、虚假报道、伪造背书或黑帽 GEO。

## 3. 目标客户

首期目标客户优先是高客单、高信任、强比较、强决策链路行业，包括：

- ToB SaaS / 企业服务公司
- 财税 / 法律 / 咨询服务公司
- 教育 / 医美 / 高客单服务机构
- 依赖搜索、问答、内容和品牌信任的企业品牌方
- 有 SEO / 内容营销 / AI 转型预算的企业

价值对象拆分：

- 付费者：企业预算方、老板、品牌方、营销负责人、增长负责人
- 决策者：老板、营销负责人、增长负责人、业务负责人
- 使用者：营销团队、内容团队、增长团队、服务交付团队
- 受益者：企业品牌方、销售团队、顾问团队
- 验收方：品牌负责人、营销负责人、业务负责人、客户验收方
- 潜在反对者：法务、合规负责人、现有 SEO/GEO 服务商、预算负责人

## 4. 核心场景

- 企业发现 DeepSeek、豆包、Qwen、元宝等 AI 平台没有推荐自己，却推荐了竞品。
- 企业发现 AI 对自己品牌、产品、服务、优势的描述不准确。
- 企业希望知道自己在 AI 问答中的提及率、推荐率、引用来源质量和竞品覆盖差距。
- 企业正在做官网改版、内容营销、SEO/GEO 升级，希望知道该补哪些内容和信任信号。
- 企业希望通过可溯源看板看到原始答案、截图、对话链接、样本范围和复测变化。

## 5. 商业结果

希望验证这些商业结果：

- 免费检测提交数和有效线索率
- 加顾问率和私域承接成交率
- GEO 体检报告成交率
- 后续优化项目转化率
- 月度代运营复购 / 续费率
- 客户看板查看率和复盘预约率
- 单客户交付工时、交付成本和真实利润

首期收入形态：

- 低价 GEO 体检报告
- 深度诊断报告
- 优化项目
- 月度代运营
- 后续工具授权或订阅候选

## 6. 可衡量指标

GEO 专属指标包括：

- AI 提及率
- 品牌推荐率
- 竞品覆盖差距
- 关键词回答覆盖率
- 引用来源质量

- 品牌描述准确率

## 7. 获客与转化路径

首期获客和转化路径：

```text
Web 引流
-> 免费 GEO 检测
-> 初筛结果
-> 添加顾问 / 私域承接
-> 顾问解读
-> 付费确认
-> 资料采集
-> 可溯源数据看板
-> GEO 体检报告
-> 顾问复盘
-> 后续优化项目 / 月度代运营
```

## 8. 资源匹配

当前可用资源和优势：

- 已有 MVP 实验经验。
- 已有同行买单信号。
- 对 GEO、SEO、AI 检索、内容营销和 AI 问答场景有前置理解。
- 可以先用人工 + 半自动方式交付，降低首期研发成本。
- 可以逐步用内部 Agent 生成问题库、分析缺口、生成报告初稿和检查风险。

需要继续验证：

- 第一批试点行业是哪 1-3 个。
- 体检报告价格区间和成交阻力。
- 获客成本是否能低于客单价。
- 单客户交付工时是否可控。
- 后续优化和代运营是否有复购。

## 9. 成本结构

需要核算：

- 获客成本
- 销售沟通成本
- 交付成本
- 人工检测和人工复核成本
- 报告制作成本
- 看板维护成本
- 客户成功和售后成本
- 合规和平台规则风险成本

初步判断：服务价值有信号，但真实利润必须扣除获客、交付、人工审核、维护、售后和风险成本后再判断。

## 10. 项目转产品判断

同行买单可以证明服务价值和项目价值有初步信号，但不能直接证明完整 SaaS 产品化成立。

需要区分：

- 项目价值：客户愿意为一次 GEO 诊断或优化项目付费。
- 服务价值：客户愿意为持续检测、诊断、优化和复盘付费。
- 产品化价值：多个行业客户愿意为标准化报告、监测、数据看板、词库和工具授权持续付费。

## 11. 低成本 MVP

MVP 必须优先验证：

- 核心用户：高客单、高信任、强比较决策行业企业
- 核心场景：AI 搜索 / AI 问答中品牌曝光不足、竞品更常被推荐、品牌描述不准确
- 核心任务：检测品牌在 AI 平台中的表现，并给出可执行优化建议
- 核心功能：免费检测、问题库、样本采集、竞品对比、证据链、可溯源看板、体检报告、复盘
- 核心交付：客户能看懂、能验收、能追溯的诊断结果和优化建议
- 核心反馈：是否愿意付费、是否愿意复盘、是否愿意继续优化或代运营

MVP 不做：

- 完整客户登录后台
- 完整 SaaS 多租户后台
- 在线支付、合同、发票系统
- 自动大规模检测平台
- 自动内容发布
- 黑帽 GEO
- 虚假评价、虚假报道、伪造背书

## 12. 反证和停止条件

- 如果客户只愿意免费检测，不愿意为体检报告或后续优化付费，商业价值不成立。
- 如果获客成本高于体检报告和后续服务收入，真实利润不成立。
- 如果交付严重依赖人工专家，Agent 无法降低成本，规模化毛利会受限。
- 如果客户只为一次性报告付费，不愿持续监测、复测、优化或代运营，SaaS 路径不成立。
- 如果 AI 平台回答波动过大且无法归因，GEO 效果难以证明。
- 如果曝光提升无法带来线索、咨询、品牌指标或客户认可，商业结果不成立。

## 13. 复判计划

下一轮复判需要收集：

- 首批免费检测提交数
- 有效线索率
- 加顾问率
- 体检报告成交率
- 报告交付周期
- 单客户交付工时
- 看板查看率
- 复盘预约率
- 后续优化 / 代运营意向
- 客户对报告和看板的认可度

成功标准：

- 有真实买单或预算确认。
- 体检报告成交率达到可接受阈值。
- 交付成本和交付周期可控。
- 客户认可报告和看板的可溯源价值。
- 至少出现后续优化、复测、代运营或转介绍信号。

失败标准：

- 只产生免费咨询，不产生付费。
- 获客、交付、维护成本无法覆盖收入。
- 客户无法理解或不认可 GEO 价值。
- 风险话术无法控制，容易变成承诺 AI 推荐或排名。
