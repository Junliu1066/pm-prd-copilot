# geo-service-prd - 商业拍板简要版

## 1. 一句话结论
值得继续做，但只值得先做服务化 MVP；不值得现在直接做完整 SaaS 或重研发自动化平台。

## 2. 当前决策面板
| 项目 | 结论 |
|---|---|
| 值不值得做 | 值得测试 |
| 统一主路由 | 服务化 MVP PRD |
| 推荐路径 | 服务化 MVP PRD |
| 当前门禁 | B：服务化 MVP |
| 证据充分性 | 足够支持服务化 MVP |
| 证据复核等级 | S_claimed（用户声明，待复核） |
| ROI 状态 | ROI 缺关键输入 |
| 是否允许完整 PRD | 否 |
| 投资门禁 | ROI 缺关键输入 |

## 3. 为什么是这个结论
| 依据 | 当前判断 | 对决策的影响 |
|---|---|---|
| 真实买单 / MVP | 用户声明存在，但仍是 S_claimed | 支持服务化 MVP，不支持直接完整 SaaS |
| 外部市场 / 竞品 | 已有公开来源和竞品参照 | 证明赛道存在，不证明本项目 ROI |
| 经营输入 | 客单价、获客成本、交付工时缺失 | ROI 暂不可计算 |
| 产品化证据 | 多客户、复购、边际成本下降未验证 | 完整 SaaS / 重研发暂缓 |

## 4. 现在可以做 / 暂缓 / 禁止
| 类型 | 内容 |
|---|---|
| 可以做 | 服务化 MVP, 客户项目验证, PRD 小修订 |
| 暂缓做 | 完整 SaaS PRD, 自动化平台建设 |
| 禁止做 | 承诺 AI 推荐结果, 黑灰产 GEO, 无验证直接重研发 |

## 5. 最需要补的证据
| 优先级 | 材料 | 当前状态 | 用来判断什么 |
|---|---|---|---|
| P0 | 付款 / 合同证据 | 缺失 | 真实买单 / 预算 / 采购流程 / S_claimed 复核 |
| P0 | 客户 / 行业记录 | 缺失 | 目标客户清晰 / 决策 / 付费对象清晰 |
| P0 | MVP 实验记录 | 缺失 | 服务闭环已试跑 / 核心价值链路可复判 |
| P0 | 复购 / 续费 / 转介绍记录 | 缺失 | 持续价值 / LTV / 产品化升级候选 |
| P1 | 报价 / 客单价证据 | 缺失 | 收入输入 / 最低利润条件 |
| P1 | 交付工时证据 | 缺失 | 交付成本 / 边际成本判断 |
| P1 | 获客来源证据 | 缺失 | 冷启动来源 / CAC/LTV 复判输入 |

## 6. 本轮建议怎么做
| 经营项 | 建议 |
|---|---|
| 建议打法 | 2-4 周服务化 MVP，不做完整 SaaS。 |
| 验证周期 | 2-4 周 |
| 最小收入信号 | 真实付款 / 合同 / 明确预算; 至少拿到第二个付费信号或明确复购 / 复测意愿 |
| 承接包状态 | 完整 |

## 7. 升级条件
- 连续出现相似客户需求
- 单客户交付工时下降到可控范围
- 客户愿意复测 / 续费 / 转介绍
- GEO 指标有可解释变化
- 核心流程可标准化为工具或看板

## 8. 停止 / 降级条件
- 2-4 周内拿不到第二个付费信号
- 客户只愿免费试用不愿付费
- 获客成本高于客单价
- 单客户交付工时持续过高
- GEO 效果无法归因

## 9. 服务化 MVP 验收重点
- 成功：免费检测提交数达到本轮验证目标
- 成功：有效线索率可被记录并解释
- 成功：加顾问率可被记录并解释
- 成功：体检报告成交率可被记录并解释
- 成功：复盘预约率可被记录并解释
- 成功：至少出现第二个付费信号或明确复测 / 续费 / 后续优化 / 代运营意向
- 失败：客户只愿免费检测但不愿付费体检报告
- 失败：报告结论不被客户认可
- 失败：单客户交付工时持续过高
- 失败：GEO 指标变化无法归因
