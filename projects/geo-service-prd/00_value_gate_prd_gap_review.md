# GEO 前置门禁 vs 现有 PRD 差距审查

- 审查对象：`00_value_gate.json`、`00_value_gate.md`、`01_prd.md`
- 审查目的：验证产品价值门禁是否能决定 GEO PRD 走向，并检查现有 PRD 是否吸收前置门禁输出
- 审查结论：建议进入 **GEO PRD V1 小修订**，不是重构

---

## 1. 前置门禁验收结论

| 项目 | 结果 |
|---|---|
| 门禁版本 | `0.3.0` |
| decision_gate | `A_ENTER_PRD` |
| value_judgment_passed | 是 |
| execution_status | `pending_human_confirmation` |
| 是否允许完整 PRD | 否，价值判断通过但人工确认未完成 |
| 证据等级 | `S` |
| 付费证据等级 | `5` |
| 风险等级 | `controlled` |
| 是否需要人工确认 | 是，原因是对外销售、价格、试点行业和客户承诺边界需要人工拍板 |

本轮先发现一个 V0.2 规则问题：反证和停止条件里的“如果……利润不成立 / SaaS 路径不成立”被误判为当前负面事实，导致第一轮错误输出 `F_NOT_RECOMMENDED`。V0.3 已修正为商业硬门禁：条件式反证不再直接触发停止结论，但仍保留为复判和 PRD 风险输入；同时 `A_ENTER_PRD` 不再等于可直接进入完整 PRD，人工确认未完成时必须保持 `pending_human_confirmation`。

---

## 2. 前置门禁给 PRD 的核心输入

| 维度 | 前置门禁输出 |
|---|---|
| 产品方向 | 对外商业产品，但第一阶段优先验证服务产品和报告交付闭环，不直接证明完整 SaaS 成立。 |
| 核心证据 | 已做 MVP 实验；同行已经为 GEO 服务买单；存在强付费信号。 |
| 核心价值 | 帮企业提升 AI 检索 / AI 问答中的品牌提及、品牌推荐、竞品对比表现和内容可引用性。 |
| GEO 指标 | AI 提及率、品牌推荐率、竞品覆盖差距、关键词回答覆盖率、引用来源质量、多模型一致性、复测变化、线索转化归因。 |
| 真实利润判断 | 初步成立，但必须继续核算获客、交付、人工审核、维护、售后、合规和风险成本。 |
| 项目转产品判断 | 项目价值和服务价值成立；产品化价值是候选，不能用同行买单直接证明 SaaS 必然成立。 |
| 反证条件 | 只免费不付费、获客成本高于客单价、交付高度依赖专家、客户不愿持续订阅、AI 平台波动无法归因。 |
| 复判材料 | 客户证据、指标基线和结果、成本核算、交付复盘、风险记录。 |
| 硬门禁状态 | 内容输入已达 PRD 标准，但价格、行业、客户承诺和是否进入正式 PRD 仍需人工确认。 |

## 3. 现有 PRD 已吸收的部分

| 维度 | 现有 PRD 状态 |
|---|---|
| 不直接做完整 SaaS | 已明确：MVP 不是完整 SaaS，也不是自动检测平台。 |
| 服务产品闭环 | 已覆盖 Web 引流、免费检测、私域成交、资料采集、看板、报告、复盘。 |
| 事实 / 假设分离 | 已有 `已确认事实`、`当前假设`、`开放问题`。 |
| GEO 指标体系 | 已有品牌提及率、品牌推荐率、引用率、竞品出现率、竞品压制率、描述准确率、证据完整度和综合评分。 |
| 风险边界 | 已有禁止承诺推荐、排名、操控 AI 答案、固定线索量等口径。 |
| 状态机和验收 | 已有 Lead / Project 状态机、成功指标、失败条件、验收 Checklist。 |
| 数据看板定位 | 已明确可溯源看板，不做完整客户登录后台。 |

## 4. 差距清单

| 优先级 | 差距 | 风险 | 建议 |
|---|---|---|---|
| P0 | PRD 没有正式引用 `00_value_gate` 作为前置依据。 | 后续 PRD 修订可能绕过价值门禁，事实、假设、反证无法被追溯。 | 在摘要后增加“前置价值门禁结论”小节，引用 `A_ENTER_PRD`、`pending_human_confirmation`、证据等级、付费等级和人工确认项。 |
| P0 | “已做 MVP 实验”和“同行已经买单”没有作为明确事实进入 PRD。 | PRD 价值判断证据不完整，无法解释为什么允许进入完整 PRD。 | 在已确认事实中补两条，但必须写清“证明服务价值有信号，不等于证明 SaaS 必然成立”。 |
| P0 | 真实利润判断还不够显性。 | 现有 PRD 有成功指标，但没有把获客成本、交付成本、人工审核成本、维护成本和售后成本连到真实利润。 | 增加“真实利润与成本假设”表，先做轻量核算字段，不做复杂财务模型。 |
| P1 | 项目价值、服务价值、产品化价值没有独立拆开。 | 容易把同行买单误读成完整平台或 SaaS 已成立。 | 增加“项目转产品判断”小节，明确先验证服务产品，再观察标准化和工具化。 |
| P1 | 反证和停止条件分散在成功指标 / 失败条件里。 | 读者不容易看到哪些情况应该停止、降级或转向。 | 增加“反证与停止条件”小节，承接 value gate 的反证清单。 |
| P1 | 价值归因口径不足。 | GEO 指标改善可能来自内容资产、AI 平台变化、品牌声量或销售动作，不能全算产品贡献。 | 在指标章节补“归因边界”：记录产品动作、交付动作、外部干扰和检测批次。 |
| P1 | 资源匹配和获客假设不够集中。 | PRD 有路径，但没有说清第一批客户、触达方式、信任来源和 CAC/LTV 风险。 | 在商业路径或上线前拍板项中补“首批客户与获客假设”。 |
| P2 | 价值对象角色可以更清楚。 | 付费者、决策者、使用者、验收方和反对者混在业务描述里。 | 增加一张价值对象矩阵，帮助后续销售、页面和原型设计。 |
| P2 | PRD 仍未显式展示价值门禁中的“人工确认项”。 | 容易让读者误以为价值通过后即可直接正式开发或对外销售。 | 在上线前人工拍板项中补：客户付款意向真实性、是否允许进入完整 PRD、价格 / 行业 / 客户承诺边界。 |

---

## 5. 推荐下一步

推荐进入 **GEO PRD V1 小修订准备**，但在人工确认前不要把它当成正式完整 PRD 执行。修订范围只覆盖：

1. 增加前置价值门禁结论。
2. 补齐 MVP 实验和同行买单事实，但避免夸大成 SaaS 必然成立。
3. 增加真实利润与成本假设表。
4. 增加项目转产品判断。
5. 增加反证与停止条件。
6. 补价值归因边界和首批获客假设。

暂不建议重构 PRD。现有 PRD 主体方向是对的，问题主要是没有把 `value_gate` 的商业判断显性接入。

---

## 6. 对价值门禁模块的反哺

这次 GEO 实战验收说明 V0.3 的方向更接近商业可用，但还需要继续观察：

- 条件式反证不能直接触发 `F_NOT_RECOMMENDED`，已在 V0.3 修复。
- `A_ENTER_PRD` 与 `can_enter_full_prd` 已分离：价值可通过，但人工确认未完成时正式 PRD 仍被阻断。
- 目标用户抽取已把 `法务` 从核心用户移到潜在反对者 / 审核角色。
- 后续可继续观察“事实 / 假设 / 反证 / 开放问题”分类质量。
- 目前不需要新增 skill / harness，不需要联网证据，也不需要复杂财务模型。
