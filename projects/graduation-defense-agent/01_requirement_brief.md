# 需求摘要：毕业答辩辅导智能体

## 一句话摘要

面向本科及研究生毕业生，提供一个基于论文材料和答辩问题库的智能模拟答辩工具，帮助学生提前发现表达、逻辑、证据和反思能力上的薄弱点，并通过追问、评分、示范改写和复练提升答辩准备效率。

## 显性需求

- 智能体不能只机械提问，需要形成“提问 → 判断回答质量 → 找出薄弱点 → 给出改写示范 → 再次追问 → 模拟真实答辩压力”的闭环。
- 问题库需要按论文类型、学生回答质量、答辩阶段动态抽取。
- 每个问题需要包含主问题、追问、考察点、回答要点和常见失误。
- 回答评分至少覆盖熟悉度、逻辑性、证据意识、反思能力、表达能力。
- 需要支持基础核验、理解深挖、方法证据、压力反驳、改进提升等训练阶段。

## 推断需求

- 学生需要导入论文题目、摘要、目录、核心章节或 PPT 结构，系统才能做个性化提问。
- MVP 应优先支持文字练习，语音练习、视频模拟和导师端可以后置。
- 产品需要明确学术诚信边界，辅助学生准备表达与应答，不替学生虚构论文内容。

## 主要未知

- 首发平台是否限定为 Web/H5、小程序、App 或学校内部系统。
- 是否需要接入学校论文规范、专业分类和学院答辩流程。
- 是否需要支持语音输入、计时、录音回放和情绪/语速分析。
- 是否面向 C 端学生付费，还是面向高校/机构 B 端采购。

## Source Trace

- `text/.md/答辩.md`: 用户提供的答辩问题库、追问规则、评分维度、回答模板。
- 外部网页信号：用于竞品外延和市场现状判断，未验证真实性，需人工核验。
