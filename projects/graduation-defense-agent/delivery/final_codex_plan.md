# 最终 Codex 执行文档：答辩训练平台化

## 1. 阶段目标

最终阶段目标是把产品升级为答辩训练平台。用户可见效果是：个人可进行更真实的多模态训练，导师/机构/学校可在授权和脱敏前提下做质量管理。

## 2. 本期输入

- 一期个人训练闭环。
- 二期个人效率增强。
- 三期导师/机构协作。
- 全量原型图中的 P16-P18 页面。
- 商业验证、学校合作、隐私评估和模型成本评估。

## 3. 本期范围

包含候选能力：

- 语音训练。
- 视频模拟。
- 学校规范库。
- 个性化知识图谱。
- 组织质量看板。
- 多端入口。

不一次性承诺全部实现；最终阶段必须按商业验证结果继续拆分子阶段。

## 4. 总体框架

| 层 | 最终阶段内容 |
| --- | --- |
| 前端 | 多端入口、语音/视频训练、组织看板 |
| 后端 | 多模态会话、规范库、画像、组织分析 |
| 数据 | MediaSession、SchoolRuleSource、LearnerGraph、OrgQualityMetric |
| AI / Prompt / RAG / Memory | 多模态评价、学校规范 RAG、长期学习画像 |
| Skill / MCP | 学校规范采集、知识库管理、多模态评审 Skill/MCP |
| Harness | source_trace、privacy、model cost、memory isolation |
| GitHub / PR | 按子阶段拆 PR |
| 教学 / 记忆 | 长期画像和组织学习必须可解释、可删除、可授权撤回 |

## 5. 页面与交互

| 页面 | 用户动作 | 系统反馈 | 异常状态 |
| --- | --- | --- | --- |
| 语音训练 | 口头回答 | 转文字、语速和结构反馈 | 权限拒绝时回到文字 |
| 视频模拟 | 进入模拟答辩 | 回放和答题复盘 | 不做情绪绝对判断 |
| 学校规范库 | 选择学校/学院规则 | 规范化训练提示 | 来源缺失不可使用 |
| 组织看板 | 查看质量趋势 | 脱敏汇总 | 样本过少不展示 |

## 6. 服务与数据

| 服务 | 数据对象 | 说明 |
| --- | --- | --- |
| 多模态服务 | MediaSession | 语音/视频训练会话 |
| 规范库服务 | SchoolRuleSource | 学校规范和来源 |
| 画像服务 | LearnerGraph | 长期能力画像 |
| 组织分析服务 | OrgQualityMetric | 脱敏组织质量指标 |

## 7. AI / Prompt / RAG / Memory

- 语音转写、口头表达反馈、视频回放分析需要单独模型评估。
- 学校规范库必须使用 RAG，并保留来源、时间和人工核验状态。
- 长期记忆分为个人训练历史、项目偏好、组织统计，不能混用。
- 用户必须能删除个人训练数据和长期画像。

## 8. Skill / MCP 接入

| 类型 | 最终策略 |
| --- | --- |
| Skill | 学校规范采集、规范库审计、多模态训练评审、组织看板质检 |
| MCP | GitHub、浏览器、学校规范源、数据库、文件存储按需接入 |
| Source trace | 学校规范和外部资料必须有来源、时间、可信度、人工核验 |
| Harness | 增加多模态隐私、RAG source trace、memory isolation、cost audit |

## 9. Codex 任务包

| task_id | goal | allowed_write_paths | forbidden_write_paths | validation | human_confirmation_points |
| --- | --- | --- | --- | --- | --- |
| PF-T1 | 语音训练子阶段 | `app/media/**`, `web/**` | 未授权录音存储路径 | 权限和删除测试 | 语音权限 |
| PF-T2 | 学校规范库 | `app/rag/**`, `app/services/rules*` | 无来源规范数据 | source trace 检查 | 规范来源 |
| PF-T3 | 长期画像 | `app/services/profile*`, `app/data/profile*` | memory-cache 跨项目路径 | 删除和解释测试 | 画像策略 |
| PF-T4 | 组织看板 | `app/services/org_metrics*`, `web/**` | 个人论文全文路径 | 脱敏检查 | 组织指标 |

## 10. 人工确认点

- 是否采集语音或视频。
- 学校规范库来源是否合法可靠。
- 长期画像保留周期。
- 组织看板脱敏规则。
- 模型成本和供应商。
- 是否接入外部 MCP。

## 11. GitHub / 发布流程

- 最终阶段不能单 PR 全量实现，必须拆成子阶段分支。
- 每个子阶段单独 PR、单独隐私评审、单独回滚方案。
- 多模态能力默认灰度发布。

## 12. Harness / 审计 / 回归

- source_trace 检查。
- memory isolation 检查。
- privacy/deletion 检查。
- model cost 检查。
- random audit 抽查外部来源和组织看板脱敏。
- 前三期主路径回归。

## 13. 教学与记忆沉淀

- 多模态反馈策略进入 open lesson，不能直接改通用 Skill。
- 学校规范更新进入来源库，不进入用户个人记忆。
- 组织统计不能反写个人评价，除非用户授权并有明确解释。
- 长期画像策略需要人工审批。

## 14. 风险与回滚

| 风险 | 回滚 |
| --- | --- |
| 语音/视频隐私风险 | 关闭多模态入口，回到文字训练 |
| 学校规范来源不可靠 | 下线该来源，回到通用规范 |
| 长期画像误导用户 | 暂停画像解释，只保留训练历史 |
| 组织看板泄露 | 关闭组织看板 |

## 15. 验收标准

- 多模态权限、删除和回滚可用。
- 学校规范有来源和更新时间。
- 长期画像可解释、可删除。
- 组织看板默认脱敏。
- 一期、二期、三期核心路径不回归。

## 16. 下一期衔接

最终阶段本身应继续拆为多个子项目，不再向单一“大版本”堆功能。每个子项目都要重新生成对应的 Codex 半自动开发文档和任务包。
