# 三期 Codex 执行文档：导师与机构协作

## 1. 阶段目标

三期目标是从个人训练扩展到辅导协作。用户可见效果是：学生授权后，导师可查看训练报告；机构或班级可查看训练完成率和薄弱模块汇总。

## 2. 本期输入

- 一期 MVP 主链路。
- 二期历史趋势和专项复练数据。
- PRD 产品包和功能矩阵同步拆分文件中的导师端、班级训练、批量报告。
- 全量原型图中的 P12-P15 页面。
- 用户对隐私授权和报告共享的反馈。

## 3. 本期范围

包含：

- 多角色账号。
- 学生授权导师查看报告。
- 导师创建训练任务。
- 班级训练完成情况。
- 批量薄弱模块报告。
- 机构后台基础统计。

不包含：

- 学校教务系统深度集成。
- 视频模拟。
- 商业支付系统。

## 4. 总体框架

| 层 | 三期内容 |
| --- | --- |
| 前端 | 导师端、班级任务、批量报告、授权管理 |
| 后端 | 组织、角色、授权、班级训练、批量统计 |
| 数据 | Organization、ClassGroup、RoleBinding、StudentConsent、ClassReport |
| AI / Prompt / RAG / Memory | 导师评语辅助、批量薄弱点摘要 |
| Skill / MCP | 可评估 GitHub/Notion/学校文档 MCP，但默认不接 |
| Harness | 权限、脱敏、导出审计 |
| GitHub / PR | 独立三期 PR |
| 教学 / 记忆 | 导师端规则不写入学生个人长期记忆 |

## 5. 页面与交互

| 页面 | 用户动作 | 系统反馈 | 异常状态 |
| --- | --- | --- | --- |
| 授权页 | 学生授权导师查看报告 | 展示授权范围 | 未授权不可查看 |
| 导师首页 | 创建训练任务 | 班级任务生成 | 班级为空提示导入 |
| 学生报告 | 导师查看授权报告 | 展示训练摘要 | 论文全文默认隐藏 |
| 批量报告 | 查看班级薄弱点 | 展示模块分布 | 数据不足提示样本过少 |

## 6. 服务与数据

| 服务 | 数据对象 | 说明 |
| --- | --- | --- |
| 组织服务 | Organization、ClassGroup | 机构和班级 |
| 权限服务 | RoleBinding、StudentConsent | 角色与授权 |
| 班级训练服务 | ClassTrainingTask | 批量创建任务 |
| 批量报告服务 | ClassReport | 汇总弱项和完成率 |

## 7. AI / Prompt / RAG / Memory

- AI 可辅助生成导师评语，但必须保留人工编辑。
- 批量报告只做脱敏总结。
- 不把导师评语直接写入学生长期画像。
- 不接学校规范 RAG，除非进入最终阶段并有来源。

## 8. Skill / MCP 接入

| 类型 | 三期策略 |
| --- | --- |
| Skill | 可新增 `class-training-report-reviewer` 提案 |
| MCP | 默认不接；机构系统接入需要单独审批 |
| Harness | 增加授权、脱敏、导出检查 |

## 9. Codex 任务包

| task_id | goal | allowed_write_paths | forbidden_write_paths | validation | human_confirmation_points |
| --- | --- | --- | --- | --- | --- |
| P3-T1 | 多角色和授权模型 | `app/auth/**`, `app/services/permission*` | `projects/**/02_prd.generated.md` | 未授权不可查看 | 权限模型 |
| P3-T2 | 导师端任务 | `app/services/class_training*`, `web/**` | 学生论文全文导出 | 创建班级任务 | B 端范围 |
| P3-T3 | 批量报告 | `app/services/class_report*` | 个人敏感资料路径 | 报告脱敏 | 脱敏规则 |
| P3-T4 | 导出与审计日志 | `app/services/audit*`, `app/services/export*` | 无授权导出路径 | 权限日志完整 | 导出权限 |

## 10. 人工确认点

- B 端是否进入本项目范围。
- 导师能看到哪些学生数据。
- 导出格式和脱敏规则。
- 机构后台是否需要收费或账号体系。

## 11. GitHub / 发布流程

- 分支：`codex/graduation-defense-phase-3`
- PR 必须附权限和隐私检查说明。
- 发布前需验证学生撤销授权。

## 12. Harness / 审计 / 回归

- 权限检查。
- 脱敏检查。
- 导出审计。
- 一期/二期主路径回归。
- Random audit 抽查授权越界。

## 13. 教学与记忆沉淀

- 导师端权限规则属于通用协作产品经验，可进入 open lesson。
- 某机构定制偏好只进项目或机构级 preference cache，不进入通用 Skill。
- 学生个人训练反馈不能被导师端规则覆盖。

## 14. 风险与回滚

| 风险 | 回滚 |
| --- | --- |
| 权限泄露 | 关闭导师端入口 |
| 批量报告误解学生能力 | 改为只显示完成率 |
| 导出敏感信息 | 暂停导出 |

## 15. 验收标准

- 未授权导师不能查看报告。
- 学生可撤销授权。
- 批量报告默认脱敏。
- 导出行为有日志。
- 一期、二期个人训练主路径不受影响。

## 16. 下一期衔接

三期输出组织、班级、授权和批量报告能力，为最终阶段的平台化、学校规范库和组织质量看板提供基础。
