# MVP 发布计划与研发任务包

- 文档状态：Draft
- 计划口径：8 周完成可内测 MVP
- 最后更新：2026-04-26

---

## 1. MVP 目标

MVP 交付一个可给内测用户使用的 Web 投研工作台，核心路径必须闭环：

1. 用户登录。
2. 查看高频事件与行情复盘。
3. 打开事件详情，看到来源、AI 摘要、相关股票和概念。
4. 从事件或概念跳转到个股详情。
5. 查看个股深度分析、动态、财务、公司档案。
6. 关注股票/概念/事件。
7. 收到站内提醒。
8. 管理员能审核 AI 风险内容和数据源状态。

## 2. 版本范围

### 2.1 MVP 必做

- 基础账号与权限。
- 高频跟踪：事件中心、事件列表、事件详情、事件日历。
- 概念中心：列表、搜索、详情、相关股票。
- 行情复盘：市场概览、热力图、涨停板块分析、概念异动。
- 个股详情：头部信息、深度分析、股票行情、概念板块、动态跟踪、财务全景、公司档案。
- AI 投研：事件摘要、概念解释、个股深度分析、来源引用、风险拦截。
- 关注与站内提醒。
- Admin：数据源、采集任务、AI 审核队列、操作日志。
- 埋点、监控、灰度、回滚。

### 2.2 MVP 可延期

- 原生移动 App。
- 团队空间和多人协作。
- 报告导出和水印。
- 复杂回测。
- 邮件、短信、微信提醒。
- 用户社群和 UGC。
- 真实交易。

| Week 6 | AI 与安全 | RAG、摘要生成、引用、安全审核、Admin 队列 | AI 可以生成有来源、有置信度、有风险提示的研究内容，高风险表达被拦截 | AI 内容有来源，违规内容被拦截，管理员可审核留痕 |
| Week 7 | 关注提醒与埋点 | Watchlist、站内提醒、埋点、看板 | 用户可以持续跟踪股票/概念/事件；产品团队能看到核心漏斗和质量指标 | 关注和提醒闭环，数据可查询 |

## 5. 研发任务包

### Task 0：项目初始化

- 目标：搭建可开发、可测试、可部署的基础工程。
- 输入文件：PRD、开发文档。
- 预期输出：
  - 前端、后端、worker、infra 目录。
  - Docker Compose。
  - 数据库迁移框架。
  - CI 基础检查。
- 验收命令：
  - 前端 lint 通过。
  - 后端单测通过。
  - 本地 `docker compose up` 后 `/health` 返回正常。
- 人工确认点：技术栈和部署环境。

### Task 1：账号、权限与基础布局

- 目标：实现登录态、角色权限、主导航和右侧快捷栏。
- 预期输出：
  - 用户表、登录接口、当前用户接口。
  - Web 主布局：顶部导航、搜索框、右侧快捷栏。
  - Free / Pro / Admin 权限判断。
- 验收标准：
  - 未登录用户访问受限功能会被引导登录。
  - Admin 菜单只对管理员展示。

### Task 2：数据源与基础实体

- 目标：实现股票、概念、原始文档的导入和查询。
- 预期输出：
  - `stocks`、`concepts`、`sources`、`raw_documents` 表。
  - 样例数据导入脚本。
  - 股票/概念/文档搜索接口。
- 验收标准：
  - 能通过名称、代码、关键词搜索股票和概念。
  - 原始文档能保留来源、发布时间、授权状态。

### Task 3：事件引擎

- 目标：把原始文档结构化为事件，并关联股票/概念。
- 预期输出：
  - 事件去重、实体链接、评分逻辑。
  - 事件列表、事件详情、事件日历 API。
  - 事件状态：draft / published / review_required / hidden。
- 验收标准：
  - 每条发布事件至少有一个来源。
  - 事件能展示重要度、情绪、置信度和关联实体。
  - 低置信度或传闻事件进入待审核。

### Task 4：高频跟踪前端

- 目标：实现高频跟踪主路径。
- 预期输出：
  - 事件中心指标卡。
  - 事件日历。
  - 实时要闻列表。
  - 事件详情页/弹窗。
  - 筛选器和排序。
- 验收标准：
  - 时间、行业、重要度筛选可用。
  - 事件详情可跳转股票和概念。
  - loading / empty / error / stale 状态完整。

### Task 5：概念中心

- 目标：实现概念检索、列表、详情和相关股票。
- 预期输出：
  - 概念中心页面。
  - 概念详情页。
  - 相关股票列表。
  - 历史时间轴。
- 验收标准：
  - 搜索概念能返回结果。
  - 概念详情展示定义、热度、事件、股票、风险。
  - Pro 内容有权限控制。

### Task 6：行情复盘

- 目标：实现市场全景和复盘视图。
- 预期输出：
  - 市场概览 API。
  - 板块热力图、板块关系图、板块分布、词云。
  - 概念异动监控。
  - 涨停板块分析。
- 验收标准：
  - 复盘页能从图表跳转到概念和股票。
  - 图表展示数据更新时间。
  - 行情数据不可用时可降级。

### Task 7：个股详情

- 目标：实现个股研究工作台。
- 预期输出：
  - 个股头部卡片。
  - 深度分析、股票行情、概念板块、动态跟踪、财务全景、公司档案六个标签。
  - K 线、财务图表、股东表格。
- 验收标准：
  - 股票搜索后进入详情页。
  - 六个标签切换不丢上下文。
  - 数据来源、更新时间、延迟状态清晰。

### Task 8：AI / RAG 与内容安全

- 目标：生成可追溯、可审核的 AI 投研内容。
- 预期输出：
  - AI 生成任务队列。
  - 事件摘要、概念解释、个股深度分析 Prompt。
  - 来源引用检查。
  - 安全规则引擎。
  - AI 审核队列。
- 验收标准：
  - AI 输出必须带来源、置信度和生成时间。
  - 买卖建议、收益承诺、无来源事实被拦截。
  - 模型超时时可降级。

### Task 9：关注与提醒

- 目标：实现用户持续跟踪。
- 预期输出：
  - Watchlist 表和 API。
  - 关注/取消关注组件。
  - 站内通知列表。
  - P0/P1 事件提醒规则。
- 验收标准：
  - 用户关注对象后右侧栏可见。
  - 关注对象有新高优先级事件时生成通知。
  - 取消关注后不再通知。

### Task 10：Admin 与运营后台

- 目标：提供数据源、任务、审核和日志管理。
- 预期输出：
  - 数据源列表和授权状态。
  - 任务监控。
  - AI 审核队列。
  - 审计日志。
- 验收标准：
  - 管理员能处理待审内容。
  - 每个审核动作有留痕。
  - 普通用户不能访问后台接口。

### Task 11：埋点、监控与发布

- 目标：保证内测可观测、可回滚。
- 预期输出：
  - 前后端核心埋点。
  - API 延迟、错误率、AI 成功率、数据采集延迟监控。
  - 灰度发布和回滚流程。
- 验收标准：
  - 核心漏斗可查询。
  - P0 告警可触达负责人。
  - 内测发布清单全部完成。

## 6. 样例数据要求

MVP 开发阶段必须准备样例数据，避免页面空转：

- 股票：至少 100 只 A 股样例，覆盖不同市场和行业。
- 概念：至少 50 个概念，包含定义和相关股票。
- 原始文档：至少 300 条新闻/公告/研报摘要样例。
- 事件：至少 100 条结构化事件，包含不同重要度和情绪。
- 个股详情：至少 20 只股票具备完整财务、股东、动态、AI 分析样例。
- 风险样例：至少 50 条违规/边界内容，用于安全规则测试。

所有样例数据必须标注来源和是否模拟，模拟数据不得在正式环境展示为真实数据。

- 未确认授权内容不进入正式展示。

## 8. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 数据授权不清 | 无法商业化展示核心数据 | 上线前建立数据源清单，未授权只用模拟或摘要 |
| AI 生成错误 | 损害信任和合规风险 | 引用约束、置信度、审核队列、用户举报 |
| 被理解为荐股 | 合规风险高 | 禁止买卖建议、收益承诺，强化研究辅助口径 |
| 行情实时性不足 | 高频用户体验差 | 标注延迟，MVP 先定义可接受口径 |
| 页面过重 | 性能差 | 图表懒加载、接口分页、缓存 |
| 功能范围过大 | 8 周无法完成 | 严格控制 MVP，延期团队协作、导出、移动 App |
| 社群内容风险 | UGC 难控 | MVP 不开放用户荐股社区 |

## 10. 人工确认清单

启动开发前建议确认：

- 数据源供应商和预算。
- 行情延迟口径。
- 是否已有证券投资咨询相关资质。
- AI 模型供应商与合规要求。
- MVP 是否收费。
- 内测用户规模。
- 设计稿是否以现有截图为基准继续。
- 是否需要保留“AGENT 社群”入口，MVP 是否隐藏。
