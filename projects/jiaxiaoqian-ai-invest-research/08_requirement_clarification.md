## 2. 我当前的默认假设

如果你不修正，我会按下面假设继续写完整 PRD：

- 产品定位：AI 投研信息聚合与研究辅助工具，不是荐股工具，不是交易工具。
- 首期市场：A 股为主，美股/港股只作为后续扩展。
- 首期端形态：Web 端优先。
- 首期主路径：事件追踪 -> 相关概念/股票 -> 个股详情 -> AI 深度分析 -> 关注提醒。
- 商业模式：先做内测验证和 Pro 订阅预留，不在 MVP 强做支付闭环。
- 数据源：先按“已授权或可合法使用的数据源”设计，未授权内容不展示全文。
- AI 输出边界：只做研究摘要、解释、风险提示、结构化分析，不输出买卖建议、目标价、仓位、收益承诺。
- 原型保真度：PRD 阶段先出低保真线框图和页面流程图，设计阶段再出高保真。

## 3. P0 待确认问题

P0 是“不确认就可能写偏产品方向”的问题。建议你先回答这些。

### P0-1：第一批目标用户是谁？

可选方向：

- A. 高频主动投资者 / 散户研究用户
- B. 财经内容创作者
- C. 机构投研助理 / 私募研究员
- D. 投顾团队内部工具

影响：

- 如果选 A，产品要更重视易懂、热点、提醒、情绪和风险提示。
- 如果选 B，产品要更重视选题、摘要、引用、内容导出。
- 如果选 C，产品要更重视数据可信度、来源追溯、研究模板、团队协作。
- 如果选 D，合规、留痕、客户适当性、审核会变成核心能力。

我的默认建议：先选 A + B，后续再扩 C。

### P0-2：MVP 主路径到底是什么？

可选方向：

- A. 事件追踪为主：先发现发生了什么，再看影响哪些概念和股票。
- B. 个股分析为主：用户先搜股票，再看 AI 分析、财务、公告、题材。
- C. 行情复盘为主：盘后看市场主线、涨停板块和热点逻辑。
- D. 概念中心为主：围绕题材、产业链、上下游和关联股票组织。

影响：

- 主路径决定首页、导航优先级、数据模型和开发排期。
- 全部都做会导致 MVP 过大，建议确定一个主路径，其他模块服务主路径。

我的默认建议：A 事件追踪为主，B 个股分析为核心承接页。

### P0-3：产品商业模式是什么？

可选方向：

- A. 免费工具，先拿用户和数据反馈。
- B. C 端订阅 Pro。
- C. B 端机构工作台。
- D. 投顾/研究服务变现。
- E. API / 数据服务。

影响：

- 如果做 C 端订阅，Pro 权限、支付、免费额度要进 PRD。
- 如果做 B 端机构，团队、权限、导出、审计、私有化要提前设计。
- 如果做投顾服务，需要先处理资质和合规，不适合直接按普通工具开发。

我的默认建议：MVP 先按 A + B 预留，不直接做 D。

### P0-4：是否已有数据源授权？

需要确认：

- 行情数据来源。
- 新闻资讯来源。
- 公告来源。
- 研报来源。
- 财务数据来源。
- 股东数据来源。
- 融资融券、龙虎榜、资金流来源。
- 概念库来源。

影响：

- 没有授权的数据不能按商业产品直接展示。
- 数据源决定接口、缓存、刷新频率、成本和法律风险。

我的默认建议：PRD 先把数据源管理和授权状态作为 P0 后台能力。

### P0-5：AI 输出允许到什么程度？

需要确认是否允许：

- 解释事件影响。
- 总结利好/利空。
- 预测后续关注变量。
- 给出机会和风险。
- 给出个股强弱判断。
- 给出买卖建议、目标价、仓位建议。

影响：

- 如果允许买卖建议，可能触发证券投资咨询/投顾边界。
- 如果只做研究辅助，产品可以更快进入内测，但必须严格禁止交易指令。

我的默认建议：MVP 只允许研究辅助，不允许买卖建议、目标价、仓位建议、收益承诺。

### P0-6：是否面向公众开放？

可选方向：

- A. 内部自用 / 小范围白名单。
- B. 面向公众开放。
- C. 只给机构客户私有化或半私有化使用。

影响：

- 面向公众开放时，生成式 AI 服务、金融内容、用户协议、隐私政策、内容审核要求更高。
- 内部白名单可先验证数据和 AI 质量。

我的默认建议：先内部白名单内测，再决定是否公众开放。

### P0-7：MVP 是否需要“AGENT 社群”？

截图中有 AGENT 社群入口，但材料没有说明具体能力。

需要确认：

- 社群只是导航占位，还是 MVP 要做？
- 是否允许用户发帖、讨论、分享观点？
- 是否有荐股/带单/投顾风险？

影响：

- UGC 社群会显著增加审核和合规压力。

我的默认建议：MVP 暂不做社群，只保留占位或隐藏入口。

### P0-8：首期要不要做可打开的交互原型？

可选方向：

- A. PRD 内低保真线框图即可。
- B. 同时输出 HTML 低保真可点击原型。
- C. 需要高保真设计稿。

影响：

- A 适合需求评审。
- B 适合研发对齐。
- C 需要进入 UI 设计阶段，不适合和 PRD 一起快速完成。

我的默认建议：B，先做 HTML 低保真原型，高保真后置。
