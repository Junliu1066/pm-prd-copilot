# Open Lessons

Open lessons are teaching inputs that have not been accepted into stable behavior.

## LESSON-20260424-001: Market default should come from preference context

- user_feedback: 竞品这一块，我做的是中国市场，如果我没有特意和你说，那就默认中国市场。
- proposed_interpretation: The user's market default should be stored as a user/project preference and read from context; generic competitor-analysis skills must not hard-code any market.
- status: proposed
- pending_decision: User needs to approve where this default lives: user-level preference, current project cache, or current request only.

## LESSON-20260424-002: Include direct and indirect competitors across the industry track

- user_feedback: 竞品不能以垂类业务为主；只要在用户所给行业赛道里面，和产品存在直接、间接影响的，都需要列入竞品。
- proposed_interpretation: Competitor analysis should include direct tools, adjacent platforms, platform-native substitutes, content/community substitutes, offline/service substitutes, and manual workarounds when they affect user decisions or workflows.
- status: proposed
- pending_decision: User needs to approve whether this becomes a durable competitor analysis rule.

## LESSON-20260424-003: Make PRD analysis outputs visually reviewable

- user_feedback: 用户群体判断可以在第一段做矩阵图；方案和 MCP/Skill 分工需要用流程图和图标呈现。
- proposed_interpretation: Broad user analysis should include a first-screen matrix, and complex PRD analysis plans should include flowcharts with skill/MCP role boundaries.
- status: proposed
- pending_decision: User needs to approve whether visual matrices and flowcharts become default output requirements.
