# PM Principles Learned From User Coaching

1. Evidence and user-pool analysis should come before PRD drafting.
2. Do not let one example or one user segment over-constrain the analysis.
3. Product logic should start from what users want to save: time, effort, money, risk, or decision cost.
4. Competitor analysis should identify advantage and leverage, not only feature parity.
5. Core scenarios should be ranked by cost, return, frequency, time saved, and MVP speed.
